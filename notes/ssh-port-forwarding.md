# 🔐 SSH 本地端口转发 (Local Port Forwarding)

## 📝 核心概念
通过 SSH 隧道进行端口转发，可以将本地机器上的网络流量安全地加密传输，并转发给远程服务器中的特定服务。当远程机器为了安全并未对公网开放某些端口（例如内部管理面板、数据库只监听在 `127.0.0.1`）时，该机制极其有用。

## ⚙️ 常用命令与详细解析

以下这条命令用于同时建立多个 SSH 端口转发（SSH 隧道）规则，且不启动远程 Shell：

```bash
ssh -N -L 8001:127.0.0.1:8000 -L 15001:127.0.0.1:50001 <user>@<server>
```

### 详细参数剖析

*   **`ssh`**: 🐧 启动 SSH 客户端程序。
*   **`-N`**: 告诉 SSH **不要执行任何远程命令**。这在你仅仅想要建立隧道进行端口转发，而不打算在远程服务器上开启交互式 Shell 时非常有用（命令执行后终端会挂起，方便后续 `Ctrl+C` 断开隧道）。
*   **`-L 8001:127.0.0.1:8000`**: ➡️ 这是第一个本地端口转发规则。
    *   `-L`：表示“Local”（本地），代表把本地计算机上的一个端口转发到远端视角的指定地址和端口。
    *   `8001`：你**本地机器**上用来监听的端口。
    *   `127.0.0.1:8000`：数据到达远端服务器后，由远端服务器转发的目标地址和端口（此处为远端服务器自己的 `8000` 端口）。
    *   **效果**：当你在本地电脑上访问 `localhost:8001` 时，流量会通过 SSH 隧道安全地发送到远程服务器，并像是由远程服务器自己发起的一样，访问它的 `127.0.0.1:8000`。
*   **`-L 15001:127.0.0.1:50001`**: ➡️ 这是第二个本地端口转发规则，原理与上一个相同。这说明我们可以在一条命令里串联多个 `-L` 参数解决多服务端口转发需求。
    *   **效果**：访问本地 `localhost:15001` 相当于远程访问它的 `50001` 端口。
*   **`<user>@<server>`**: 👤 你的 SSH 登录凭据。`<user>` 是远程服务器上的用户名，`<server>` 是远程服务器的 IP 地址或主机名。

## 🎯 实际应用案例：安全访问局域网仅限本地 (localhost) 监听的服务

**场景描述**：
局域网内有一台目标机器（IP: `10.10.0.104`），上面运行着一个叫做 OpenClaw 的服务。该服务出于安全协议，默认配置为只能通过 `localhost` (`127.0.0.1`) 访问，无法直接通过浏览器输入其局域网 IP 去查看面板。

**解决方案（建立本地转发隧道）**：
为了能在个人电脑上通过浏览器直接浏览该内网机器上的 OpenClaw 用户界面，我们可以建立以下 SSH 隧道：

```bash
ssh -N -L 18789:127.0.0.1:18789 openclaw@10.10.0.104
```

**运行与验证效果**：
1. 本地执行命令后，终端将保持阻塞状态（由 `-N` 参数控制），并在本地机器默默监听 `18789` 端口。
2. 打开本地电脑的浏览器，访问 `http://127.0.0.1:18789` 即可显示 OpenClaw 面板。
3. 网络请求被安全地封装入 SSH 隧道送入 `10.10.0.104`，再在该机器内部被送给 `127.0.0.1:18789` 真正的服务去解析。实现**将远端内网环境安全拉回本地**的假象。
