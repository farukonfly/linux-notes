# Lab: SSH 本地端口转发 (Local Port Forwarding)

## 🎯 实验目标 (Objectives)
* 理解 SSH 隧道的概念及其在安全访问内网资源中的应用。
* 掌握 `ssh -L` (Local Port Forwarding) 命令的语法和实际操作。
* 验证本地机器如何通过安全隧道访问远端服务器上仅限本地 (127.0.0.1) 监听的服务。

## 📖 实验场景假设 (Scenario)
你拥有一台可通过 SSH 访问的远程 Linux 服务器（例如 IP 为 `10.10.0.104`）。
考虑到安全原因，该服务器上的某项核心服务（例如一个管理面板或内部 API）被配置为仅监听在本地回环地址 `127.0.0.1` 上的 `18789` 端口。外部网络无法直接通过 `http://10.10.0.104:18789` 访问它。
为了在你的个人电脑上安全地使用浏览器操作面板，你需要建立一条安全的 SSH 隧道，将本地的 `18789` 端口映射到远程服务器上的 `127.0.0.1:18789`。

## 📝 任务清单 (Tasks)

### 任务 1: 确认远程服务的状态
*目标：在实际创建隧道之前，我们需要确认远程服务器上确实有服务在监听目标端口。*
- SSH 连接到你的远程机器 (`10.10.0.104`)。
- 在远程机器上执行一个检查网络端口监听状态的命令，或者使用类似 `curl` 命令测试本地能否访问该服务。
- 取证证明目标服务确实在 `127.0.0.1:18789` 上运行。

### 任务 2: 在本地建立 SSH 隧道
*目标：在你的本地电脑（不是远程服务器）上执行端口转发命令。*
- 打开一个新的本地终端。
- 构造并执行 `ssh` 命令，将本地由于 `18789` 端口接收到的流量，转发到远程机器视角下的 `127.0.0.1:18789`。
- 确保命令在后台运行或阻塞当前终端（不执行远程 shell）。

### 任务 3: 验证服务访问
*目标：通过浏览器或命令行验证隧道是否生效。*
- 保持 SSH 隧道命令所在的终端处于运行状态。
- 打开本地浏览器，访问映射到你本地电脑上的端口。
- 观察是否成功展示了远程服务的页面。
